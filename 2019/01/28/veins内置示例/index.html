veins内置示例

发表于 2019-01-28
字数统计: 1.5k | 阅读时长 ≈ 7

内置示例解读

  • antenna.xml 对车辆的天线做出配置

  • config.xml 配置物理层无线电传播相关参数，示例中的信号路径衰落的模型配置为“SimplePathModel”，障碍物引起的信号衰落模型配置设置为”SimpleObstacleShowing”,通常情况下使用默认的即可；

  • debug 调试信息;

  • erlangen.launchd.xml 建立与sumo的通信;

  • erlangen.net.xml sumo路网文件，仿真时可以进行替换，参考sumo文档;

  • erlangen.poly.xml 如果SUMO路网文件是由地图转化而来的，如从OpenStreetMap导入的，则路网文件中还将包含所选地区的建筑物的信息，通过SUMO提供的OpenStreetMap Web接口便可生成记录这些建筑物形状、大小和位置信息的配置文件，作为config.xml内信号衰落模型的输入；如果不考虑建筑物对于信号传播的影响则无需此文件，当然config.xml内的信号衰落模型的配置也要删去;

  • erlangen.rou.xml 交通流文件;

  • erlangen.sumo.cfg SUMO运行文件，可对运行的时长等参数进行配置

  • RSUExampleScnario.ned ned文件，参考[omnetpp++文档](https://doc.omnetpp.org/omnetpp/manual/#cha:ned-lang

omnetpp.ini

network = RSUExampleScenario 关键参数，需要替换为自己的network的ned文件

  • sim-time-limit
    sim-time-limit = 200s 网络仿真的时间限制，需要与SUMO生成的交通流运行时间对应

  • RSU SETTINGS
    对RSUExampleScnario.ned中路侧单元的ned中的参数进行设置。

对RSUExampleScnario.ned中路侧单元的ned中的参数进行设置。
applType 指定Rsu的应用层，即说明RSU会做什么.applType 指定应用层的ned,而应用层的ned的功能由同名同文件夹的C++文件定义， 做应用层研究的要改变的就是这个应用层的ned与对应的C++文件

  • 11p specific parameters NIC-Settings 对MAC层ned（nic）相关参数做出设置，nic包含两个子ned（mac1609_4和phy80211p）; connectionManager基本保持不变，如果研究信号的传播范围影响需要改变maxInterDist

  • WaveApplayer 对移动的节点（车辆）的应用层做出配置，和RSU的道理相同

  • Mobility 对移动的模块进行设置，其中accidentCount,accidentStart,accidentDuration三个参数是对移动节点发生事故的事件做出设置，这是veins的example的演示功能（事故预警消息的传播），此时用不到直接设置accidentCount=0.
